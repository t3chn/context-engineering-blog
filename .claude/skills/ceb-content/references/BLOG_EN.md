# Blog Platform Guide — English

## Overview

English articles are translations of Russian originals.
Keep the same structure and meaning, but write naturally in English.

## Translation Rules

1. **Natural English** — not literal translation
2. **Same structure** — keep all sections
3. **Technical terms** — use standard English terminology
4. **Tone** — maintain the same technical, honest style

## Frontmatter

```yaml
---
title: "Article Title"
description: "SEO description, 1-2 sentences"
date: YYYY-MM-DD
tags: ["context-engineering", "topic"]
lang: en
---
```

## Common translations

| Russian | English |
|---------|---------|
| Проблема | Problem |
| Контекст | Context |
| Решение | Solution |
| Инсайт | Insight |
| Источники | Sources |

## Tone in English

- Direct and clear
- Technical but accessible
- No filler words
- Active voice preferred

## Example

**Russian:**
```
Потратил день на отладку промпта.
Проблема была не в промпте.
```

**English:**
```
Spent a day debugging a prompt.
The problem wasn't the prompt.
```

## Checklist

- [ ] Natural English, not literal translation?
- [ ] Same structure as Russian version?
- [ ] Technical terms correct?
- [ ] Frontmatter updated (lang: en)?
- [ ] Same date as Russian version?
